# Code of Conduct

This project adopts the **[Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)**
in full. The full text is maintained at the link above; this file describes
how the standard is applied here.

## Scope

The Code of Conduct applies to:

- All project spaces (this repo, issues, pull requests, discussions)
- Project communications elsewhere when an individual is officially
  representing the project (e.g., posting from a `@pi-workbench` account)

## Reporting

If you observe behaviour that violates the Contributor Covenant, contact the
maintainer privately:

- **Email:** the address listed in `git log --format="%ae" | head -1` for
  the most recent maintainer commit, OR
- **GitHub:** open a private security advisory on the repository
  (`Security → Advisories → Report a vulnerability`) — note the report is
  about conduct, not a vulnerability, and the maintainer will route it
  appropriately.

Do **not** file conduct reports as public issues. Reports should include:

1. What happened (with links / commit IDs / quoted content where applicable)
2. When and where it happened
3. Whether you've experienced or witnessed prior incidents from the same person
4. Whether you've already taken any direct steps (and if so, what)

## Response

The maintainer commits to:

- **Acknowledge** every report within 7 days
- **Investigate** before taking any action
- **Treat the reporter's identity as confidential** unless they explicitly
  consent to disclosure or the law requires it
- **Apply consequences proportional to the violation**, following the
  Contributor Covenant's "Enforcement Guidelines" ladder (correction →
  warning → temporary ban → permanent ban)
- **Communicate the outcome** to the reporter once a decision is reached

## Enforcement

The project maintainer has final authority over enforcement decisions. There
is no separate committee for a single-maintainer project of this size; if the
project grows to require one, this document will be updated.
